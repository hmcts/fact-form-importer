What this step does

This is not a {{ comparison_summary.total }}-item approval queue. The {{ comparison_summary.total }} figure is the number of planned section actions compared with live FaCT. By default, the table shows only decisions that still need a person.

  • {{ comparison_summary.empty_target }} actions target an empty live section and {{ comparison_summary.no_change }} produce no effective change; neither needs a live-data approval.
  • {{ comparison_summary.approval_required }} populated live sections would change and await approval.
  • {{ comparison_summary.conflicts }} actions have duplicate or ambiguous business types and need a resolution rather than a live-data approval.
  • High-confidence LLM values are already automatically approved. This page has no model confidence of its own: its separate approval protects populated live FaCT data from an unintended overwrite.
  • Submitted values are merged into live FaCT data. Blank and unmatched live values are preserved, and a later live or proposed change invalidates approval.
